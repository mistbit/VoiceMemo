# 文档规范

## 目标

- 文档可直接指导后续迭代与排障。
- 关键行为可以追溯到具体代码路径与数据结构。
- 避免泄露密钥与隐私信息，仅使用占位符与脱敏说明。

## 命名规范

- 文档目录：仓库根目录下的 `doc/`。
- 文件命名：
  - 英文：`NN-topic-in-kebab-case.md`
  - 中文：`NN-topic-in-kebab-case.zh-CN.md`
- 编号保持稳定；新增文档使用下一个序号追加。

## 内容规范

除非是短说明，否则建议使用以下结构：

1. 文档目的
2. 范围 / 非目标
3. 术语表
4. 高层流程（复杂流程优先用 Mermaid）
5. 核心类型与职责（映射到文件/类）
6. 错误处理
7. 安全与隐私注意事项
8. 运行与配置说明（路径、开关、参数）
9. 验证方式（测试、手动检查）

## 代码引用

- 文档内使用相对路径（例如 `Sources/VoiceMemo/AudioRecorder.swift`）。
- 引用关键函数/类型时，写出文件路径与标识符名称，方便快速定位。

## 密钥与隐私

- 不要在文档中粘贴真实的 AccessKeyId / AccessKeySecret / AppKey。
- 使用占位符，例如：
  - `AK_ID=...`
  - `AK_SECRET=...`
  - `TINGWU_APPKEY=...`
